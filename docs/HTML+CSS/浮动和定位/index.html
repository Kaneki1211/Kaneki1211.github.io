九、浮动和定位

1、浮动​

浮动的作用​

  • 让块元素从上下排列变成水平排列（所有的盒子全部设置浮动）

原理

  • 浮动的盒子会脱离文档流
  • 浮动显示的范围由父元素的宽度决定，溢出的浮动元素会换行显示
  • float：left 左浮动 right 右浮动 none 不浮动

.wrap {
  width: 300px;
  height: 400px;
  border: 1px #000 solid;
}
.box1,
.box2,
.box3 {
  width: 100px;
  height: 100px;
  border: 1px solid #000;
}
.box1 {
  float: left;
  background: red;
}
.box2 {
  float: left;
  background: yellow;
}
.box3 {
  float: left;
  background-color: green;
}

<div class="wrap">
        <div class="box1"></div>
        <div class="box2"></div>
        <div class="box3"></div>
        <div class="box1"></div>
        <div class="box2"></div>
</div>

浮动的盒子会脱离文档流
如果浮动下方是正常占位的盒子，下方的盒子就会和浮动的盒子产生重叠效果（文本，图片，表单）

文本，表单，图片不会重叠

清除浮动​

给受浮动影响的元素增加 clear 属性
chear： left right both清除两边浮动

高度塌陷​

父元素的高度由内容撑开，如果内容浮动不占位，父元素识别不了内容的高度，就会出现高度塌陷

解决高度塌陷问题

  • 1.触发 BFC 解决（浮动区域的盒子也参与高度计算）
    overflow：hidden 推荐
    display：inline-block
    float
  • 2.在所有浮动元素的最下方设置空盒子，给空盒子设置清除浮动 clear 属性，把父元素的高度撑开
  • 3.万能清除法 用 after 把父元素的高度撑开，解决高度塌陷的问题
    .clear-fix::after {
      content: "";
      height: 0;
      display: block;
      clear: both;
      overflow: hidden;
    }
    解决 IE 低版本的兼容
    .clear-fix{
    zoom：1
    }
  • 4.在浮动元素最后面，添加一个块元素，并给元素设置clear:both

万能清除法​

用 after 把父元素的高度撑开，解决高度塌陷的问题

<div class="box1 .clear-fix">
  <div class="box2"></div>
</div>
.clear-fix::after {
  /*  content属性必须要设置 */
  content: "";
  height: 0;
  display: block;
  clear: both;
  overflow: hidden;
}
.box1 {
  border: 1px solid;
}
.box2 {
  width: 400px;
  height: 600px;
  background-color: aqua;
  float: left;
}

双伪类解决​

<div class="box1 .clear-fix">
  <div class="box2"></div>
</div>
.box1::after,
.box1::before {
  content: "";
  display: block;
}
.box1::after {
  clear: both;
}

.box1 {
  border: 1px solid;
}
.box2 {
  width: 400px;
  height: 600px;
  background-color: aqua;
  float: left;
}

2、定位​

定位元素可以触发五个属性

  • 4 个是位置方向 left right top bottom
  • 1 个层级关系 z-index:1 仅支持整数，可以设置负值

设置定位 position​

  • 相对定位 relative
    • 占位
    • 相对于自身的原始位置
  • 绝对定位 absolute
    • 不占位，脱离文档流（完全脱离）
    • 找最近有定位属性的祖先元素，再找的就是浏览器的窗口
  • 固定定位 fixed
    • 不占位，脱离文档流（完全脱离）
    • 固定定位始终以浏览器窗口为位置移动的参照物
  • 粘性定位 sticky
    • 占位
    • 设置粘性定位，必须配合位置方向一起设置，如果没有方向粘性定位就会失效
    • 粘性定位的父元素不能设置 overflow 属性，否则粘性定位会失效
  • 取消定位 static

定位居中​

第一种

/* 父元素 */
section {
  float: left;
  width: 400px;
  height: 400px;
  border: solid;
  position: relative;
}
/* 子元素 */
.box1 {
  width: 100px;
  height: 100px;
  background-color: #bfa;
  position: absolute;
  left: 0;
  right: 0;
  top: 0;
  bottom: 0;
  margin: auto;
}

第二种

/* 父元素 */
section {
  float: left;
  width: 400px;
  height: 400px;
  border: solid;
  position: relative;
}
/* 子元素 */
.box2 {
  width: 100px;
  height: 100px;
  background-color: blueviolet;
  position: absolute;
  top: 50%;
  left: 50%;
  margin-top: -50px;
  margin-left: -50px;
}
